# CodeReviewerBot 需求文档

## 1. 项目概述

一个基于 AI 的代码审查助手，自动分析 GitHub PR 并提供审查意见。

## 2. 核心功能需求

### 2.1 GitHub PR 自动审查
- 监听指定 GitHub 仓库的 Pull Request 事件
- 自动提取 PR 的 diff 变更内容
- 对每个变更文件进行代码审查
- 在 PR 上以评论形式发布审查意见

### 2.2 代码审查能力
- 代码风格一致性检查
- 潜在 Bug 检测（空指针、资源泄露、线程安全等）
- 安全漏洞扫描（SQL 注入、XSS、CSRF 等）
- 性能建议（不必要的重复计算、N+1 查询等）
- 最佳实践建议（设计模式、代码结构等）

### 2.3 审查报告
- 按文件分类展示审查结果
- 问题严重程度分级（Critical / Warning / Suggestion）
- 每个问题附带代码行号和建议修复方案
- 支持 Markdown 格式的审查报告

### 2.4 配置管理
- 支持按仓库配置审查规则
- 可开启/关闭特定检查项
- 可配置忽略规则（路径、文件类型等）

## 3. 技术栈要求

- 语言：Python 3.11+
- Web 框架：FastAPI
- AI 模型：Claude API（Anthropic）
- 数据库：PostgreSQL（生产）/ SQLite（开发）
- 任务队列：Celery + Redis
- 部署：Docker

## 4. 非功能需求

- 审查响应时间 < 5 分钟（从 PR 创建到评论发布）
- 支持并发处理多个 PR
- 审查结果可追溯、可审计
